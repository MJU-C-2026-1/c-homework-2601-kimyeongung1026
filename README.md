# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
나만의 맞춤형 카페 키오스크 주문 시스템




## 2. 시나리오 (5~10줄)
이 프로그램은 카페 고객의 주문 접수부터 결제 및 영수증 출력까지 완벽하게 처리하는 스마트 카페 키오스크 시스템입니다. 사용자는 화면에 고정된 메뉴의 가격 정보를 확인하고, 대기 진동벨을 위한 고객 이니셜과 원하는 음료(아메리카노,라떼)의 주문 수량을 입력합니다. 시스템은 이를 바탕으로 사칙연산을 활용하여 세금 포함 전 주문 합계를 계산하고, 부가세(10%)를  적용하여 최종 결제 금액을 산출합니다. 향후에는 조건문을 도입하여 특정 금액 이상 구매 시 세트 할인을 적용하거나, 반복문을 통해 프로그램이 종료되기 전까지 여러 고객의 주문을 끊임없이 처리할 수 있도록 실제 키오스크와 유사한 주문 시스템을 완성할 것입니다.
<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
(예시)
1. 카페 메뉴판 및 기본 가격 화면 출력
2. 고객 정보(이니셜) 입력
3.  음료 주문 수량 입력
4. 주문 총액 및 부가세(10%) 산출
5. 최종 결제 금액 및 계산 영수증 출력

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
사용자로부터 고객 이니셜(영문1글자)과 아메리카노, 라떼의 주문 수량을 입력받아 총 결제 금액을 계산하고 영수증 형태로 출력합니다.
고객 이니셜(문자형), 수량 및 기본 금액(정수형),부가세 및 최종 결제 금액(실수형)을 입력받고 계산하는 과정을 구현했습니다.
사칙연산을 활용해 주문 합계와 10% 부가세를 더한 최종 결제 금액을 산출했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약: 현재 배운 문법(int, printf, scanf,char,double)을 활용하여 실생활에 적용할 수 있는 카페 키오스크 주문 시스템 시나리오 기획 요청
    * **적용 내용:** 아직 조건문이나 반복문을 배우지 않은 상황에 맞춰, 순차적 구조(위에서 아래로 실행)만으로 주문 입력부터 영수증 출력까지 완결성을 갖춘 V1.0프로그램 뼈대를 설계함.
      
 2. **내용 2**
    * **프롬프트 요약:** "영수증 출력 프로그램에 필요한 변수 설정 및 적절한 자료형 할당 논의"
    * **적용 내용:** 주문 수량과 메뉴 가격은 정수형(int), 고객 식별용 이니셜은 문자형(char), 부가세 (10%) 곱셈 연산 시 발생할 수 있는 소수점 처리를 위해 실수형(double)으로 분리 선언하여 데이터 구조의 안정성을 높임
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 코드 실행 시 한글(메뉴판 및 안내 문구)이 깨져서 출력되는 현상이 발생함.
     * **해결:** 콘솔의 언어 인코딩 문제임을 파악하고 AI에게 조언을 받아 <stdlib.h> 헤더를 추가한 뒤 main 함수 첫 줄에 system("chcp 65001"); 코드를 삽입하여 UTF 환경으로 전환함으로써 한글 깨짐을 완벽하게 해결함.
  2. **문제 2:** 부가세를 포함하지 않은 총 주문 금액을 계산하는 과정에서 식은 맞으나 컴파일 에러가 발생함.
     * **해결:**AI에게 코드 검토를 요청한 결과, 계산식에 사용된 subtotal 변수를 미리 선언하지 않았다는 것을 발견함. 변수 선언부에 int subtotal; 을 추가하여 해결함. C언어에서 변수 선언의 중요성을 깨달음
